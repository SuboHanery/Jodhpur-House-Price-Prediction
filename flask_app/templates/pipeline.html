Project Development Pipeline

A step-by-step overview of the methodology and technologies used to build the Jodhpur AI Predictor from scratch.

Phase 1: Data Gathering & Cleaning

We began by collecting raw real estate listings from Jodhpur. The dataset underwent rigorous cleaning to remove outliers, handle missing values, and standardize formatting.

Dataset Size: 8,000 Properties Tech: Python, Pandas

Phase 2: Feature Engineering

Raw data isn't enough for an AI. We applied One-Hot Encoding to categorical locations (like Sardarpura vs Shastri Nagar) and used a StandardScaler to normalize numerical data (like area size) so the model could process it efficiently.

Input Features: 27 Variables Tech: Scikit-Learn, NumPy

Phase 3: Model Training

We tested multiple algorithms (Linear Regression, Decision Trees) before finalizing a Random Forest Regressor. The model was trained, tested for variance, and exported into a `.pkl` file for instant real-time predictions.

Accuracy Score: 93.2% Tech: Scikit-Learn, Pickle

Phase 4: Web Deployment

To make the model accessible, we wrapped it in a secure REST API. We then designed a sleek, modern UI using a Glassmorphism aesthetic to allow end-users to input property details and receive immediate valuations without requiring programming knowledge.

Architecture: Full-Stack SPA Tech: Flask, HTML5, CSS3, JS
